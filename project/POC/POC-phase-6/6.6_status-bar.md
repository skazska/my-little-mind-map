# Task 6.6 — Status Bar Across All Apps

> Phase: 6 (Polish & Release)
> Decision: D-014
> Requirements: NFR-7.1–7.5, P6-R11

---

## Objective

Add a persistent status bar to all client apps (desktop, web, mobile) showing storage path, note/topic counts, and app version.

## Sub-tasks

| # | Sub-task | Platform | Status |
|---|----------|----------|--------|
| 6.6.1 | Desktop app (Tauri) status bar | Desktop | Done |
| 6.6.2 | Web app status bar | Web | Not started |
| 6.6.3 | iOS app status bar | iOS | Not started |
| 6.6.4 | Android app status bar | Android | Not started |

## Implementation Notes

### Desktop (Done)

- `get_storage_path` Tauri command returns `StorageHandle.root().display()` as `String`
- `StatusBar` React component with truncatable path (ellipsis), fixed counts, and version
- App version via `getVersion()` from `@tauri-apps/api/app` (requires `core:default` capability)
- Single shared layout wrapper in `App.tsx` renders StatusBar once across all views
- PR: #9 (`chore/desktop-status-panel`)

### Web (Pending)

- Storage path: show backend URL or IndexedDB/WASM storage location
- Version: read from `package.json` or inject at build time via Vite `define`
- StatusBar component: same design, adapted for web context

### iOS (Pending)

- Storage path: app sandbox documents directory
- Version: `Bundle.main.infoDictionary["CFBundleShortVersionString"]`
- StatusBar: SwiftUI view at bottom of navigation stack

### Android (Pending)

- Storage path: `context.filesDir` or app-specific storage
- Version: `BuildConfig.VERSION_NAME`
- StatusBar: Compose component at bottom of scaffold

## Design (D-014)

- Status bar pinned at bottom of window/screen, full width
- Content: storage path (left, truncatable), counts (center), version (right)
- Subtle styling: small font, muted colors, thin top border
- Must not obstruct main content — main area scrolls independently

## Acceptance Criteria

- [ ] All client apps display a persistent status bar
- [ ] Status bar shows correct storage path for each platform
- [ ] Status bar shows note and topic counts matching actual data
- [ ] Status bar shows correct app version
- [ ] Long storage paths truncate with ellipsis without breaking layout
- [ ] Status bar is visible across all views/screens
